## Design System

| Property | Value |

## Interaction Patterns (Phase 6 Scope)

### Dashboard Navigation

| Interaction | Behaviour |
|-------------|-----------|
| Click sidebar "Epic Experiment" | Navigate to dashboard. Screen builds all 7 panels. |
| Dashboard is the default landing screen | When app loads and no hash route is set, navigate to `#/` (Epic Experiment). |

### Quick Add Bar

| Interaction | Behaviour |
|-------------|-----------|
| Focus Quick Add input | Blue border ring (`border-primary`, `shadow-glow-blue`). Placeholder text clears. |
| Type card name | Debounce 150ms. Autocomplete dropdown appears below input (max 6 results). Separate from global search. |
| Select autocomplete result | Resolve card. Apply condition (from dropdown) and foil (from checkbox). Add to collection via `collection.addCard()`. Show success toast. Flash Quick Add bar background to `primary/10` for 300ms. Log activity. Clear input. |
| Press Enter without selection | Attempt to parse full input via `parseBatchLine()`. If resolved, add card. If not, show error toast. |
| Condition dropdown | Small dropdown button next to input. Options: NM, LP, MP, HP, DMG. Default NM. Persists across adds (does not reset). |
| Foil checkbox | Small toggle button next to condition. Default unchecked. Persists across adds. |

### Deck Quick-Launch Grid

| Interaction | Behaviour |
|-------------|-----------|
| Click deck tile | Navigate to `#/deck-builder/{deckId}`. Opens the three-panel editor for that deck. |
| Hover deck tile | Border shifts to `border-primary` with `shadow-glow-blue`. Commander art thumbnail brightens slightly (`brightness(1.1)`). |
| "View all" link | Navigate to `#/deck-builder` (deck landing page). |

### Activity Timeline

| Interaction | Behaviour |
|-------------|-----------|
| Scroll | Vertical scroll within the panel. Max visible height 320px with overflow-y-auto. |
| Click activity entry (card-related) | Open card detail flyout for that card. |
| Click activity entry (deck-related) | Navigate to that deck in the editor. |

### Status Chip

| Interaction | Behaviour |
|-------------|-----------|
| Online event fires | Recalculate status. If stale (> 24h), trigger silent background bulk data refresh (D-16). Update chip reactively. |
| Offline event fires | Chip immediately changes to OFFLINE state (red dot, red text). |
| Hover | Tooltip showing last update time: `Last updated: {datetime}` (label 400). |

### Undo System

| Interaction | Behaviour |
|-------------|-----------|
| Destructive action fires | Item hidden from UI immediately (optimistic). Undo entry pushed to in-memory stack. Undo toast appears with UNDO button and 10s countdown bar. |
| Click UNDO button | Restore item to UI from in-memory snapshot. Cancel 10s timer. Dismiss undo toast. Show success toast "Restored {item}." |
| Press Ctrl+Z | Undo the most recent entry in the stack (LIFO). Same restore behaviour as clicking UNDO. |
| 10s countdown expires | Commit the actual database deletion. Remove entry from undo stack. Toast fades out. |
| Multiple pending undos | Each gets its own toast and its own countdown. UNDO button on each toast targets that specific entry by ID. Ctrl+Z always targets the most recent. |

### Undo Toast Visual

| Property | Value |
|----------|-------|
| Position | Bottom-right stack (same as regular toasts) |
| Left border | 4px `secondary` (#E23838) |
| Background | `surface` (#14161C) |
| Message | Body 14px Space Grotesk 400 |
| UNDO button | Label 11px JetBrains Mono 700 uppercase, `text-primary` colour, hover `text-text-primary` |
| Countdown bar | Full width at bottom of toast, `bg-primary` (#0D52BD), shrinks from 100% to 0% over 10s, height 2px |
| Duration | 10000ms (overrides default 5000ms) |

### Keyboard Shortcuts

| Key | Behaviour | Priority |
|-----|-----------|----------|
| `/` | Focus global search input. Prevented when focus is already in an input/textarea. | Always (body level) |
| `Escape` | Close in priority order: (1) Shortcut cheat sheet modal, (2) Card detail flyout, (3) Context menu, (4) Autocomplete dropdown, (5) Clear search input. Only closes one thing per press. | Highest z-index first |
| `Ctrl+Z` | Undo most recent entry in undo stack. No-op if stack is empty. Prevented when focus is in input/textarea (browser native undo takes priority). | Body level |
| `?` | Toggle keyboard shortcut cheat sheet modal. Prevented when focus is in input/textarea. | Body level |

### Shortcut Cheat Sheet Modal

| Property | Value |
|----------|-------|
| Trigger | `?` key (body level, not from within inputs) |
| Overlay | `bg-background/80` with `backdrop-filter: blur(8px)` |
| Modal size | 480px wide, auto height, centred vertically and horizontally |
| Background | `bg-surface` |
| Border | `border border-border-ghost` |
| Layout | 2-column grid: `<kbd>` key left, description right |
| `<kbd>` styling | `bg-surface-hover border border-border-ghost px-sm py-xs font-mono text-[11px] text-text-primary` |
| Description | Body 14px Space Grotesk 400 |
| Close | Escape key, click backdrop, or press `?` again |
