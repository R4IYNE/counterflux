# Phase 4 — UI Design Contract

> Visual and interaction contract for the Intelligence Layer phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (vanilla JS + Alpine.js, no component library) |
| Preset | not applicable |
| Component library | none (custom components, Organic Brutalism system) |
| Icon library | Material Symbols Outlined (font, `FILL` 0, `wght` 400, `opsz` 24) |

| Element | Copy |
|---------|------|
| Below threshold | `{CATEGORY}: {count} CARDS -- BELOW {threshold}` (e.g. `RAMP: 3 CARDS -- BELOW 10`) |

| Settings save action | `Save Thresholds` |

### Toast Notifications (Phase 4 Actions)

| Action | Type | Copy |
|--------|------|------|
| Synergy card added | success | `Added {card name} from suggestions.` |
| EDHREC cache refreshed | info | `EDHREC data refreshed for {commander name}.` |
| EDHREC fetch failed | warning | `Intelligence unavailable -- using local heuristics.` |
| Spellbook fetch failed | warning | `Combo detection unavailable. Check your connection.` |
| Threshold saved | success | `Gap thresholds updated.` |
| Threshold reset | info | `Gap thresholds reset to defaults.` |

### Error States

| Scenario | Copy |
|----------|------|
| EDHREC network failure | `Intelligence unavailable -- using local heuristics.` (inline, not blocking) |
| Commander Spellbook failure | `Combo detection unavailable.` (inline, not blocking) |
| Both APIs fail | `External intelligence unavailable. Deck builder functions normally with local analysis.` |
| EDHREC CORS blocked (dev) | `EDHREC connection blocked. Try refreshing or check network.` |

### Destructive Actions

No destructive actions in Phase 4. Gap threshold edits are non-destructive (reset restores defaults). Synergy card additions follow existing deck add flow.

## Interaction Patterns (Phase 4 Scope)

### Synergy Suggestions

| Interaction | Behaviour |
|-------------|-----------|
| Commander selected/changed | Intelligence store fetches EDHREC data. Synergy section shows loading skeleton (3 shimmer rows). On success, populates 10-15 synergy cards sorted by lift score descending. On failure, shows fallback message. |
| Hover synergy card | Background shifts to surface-hover (#1C1F28). |
| Click synergy card | Card added to deck via `deck.addCard()`. Success toast shown. Card removed from synergy list (already in deck). |
| Scroll synergy list | Vertical scroll within fixed-height container (max 400px). No virtualisation needed (max 15 items). |
| EDHREC cache stale (7+ days) | Automatic re-fetch on next commander load. No manual refresh UI needed (refresh happens transparently). |

### Combo Detection

| Interaction | Behaviour |
|-------------|-----------|
| Deck cards change | Debounced (500ms) re-fetch to Commander Spellbook `find-my-combos`. Combo badges update on card tiles. Near-miss section updates in sidebar. |
| Hover combo badge | Tooltip: "Part of {N} combo(s)". |
| Click combo badge | Combo popover appears anchored to badge. Shows all combos this card participates in. |
| Click outside popover | Popover closes. |
| Press Escape | Popover closes. |
| Click missing piece name (near-miss) | Opens existing card detail flyout for that card. |
| Spellbook API fails | Combo badges not rendered. Near-miss section shows "Combo detection unavailable." No blocking. |

### Salt Score

| Interaction | Behaviour |
|-------------|-----------|
| Commander loaded | Salt score fetched from EDHREC commander JSON. Gauge renders immediately with commander salt. Per-card salt loaded lazily in background. Aggregate updates as individual card salt values arrive. |
| Gauge colour | Animates fill width over 300ms ease-out. Colour determined by final value: success/warning/secondary. |
| EDHREC fails | Gauge shows "SALT: N/A -- EDHREC UNAVAILABLE" in text-dim. |

### Gap Detection

| Interaction | Behaviour |
|-------------|-----------|
| Deck cards change | Gap detection runs synchronously (pure function, no API call). Tag breakdown rows update immediately. Warning icons appear/disappear as counts cross thresholds. |
| Click gear icon | Threshold settings popover opens anchored to icon. |
| Edit threshold value | Number input, min 0, max 99. No live preview -- changes applied on "Save Thresholds". |
| Click "Save Thresholds" | Thresholds saved per-deck in Dexie. Popover closes. Success toast. Gap warnings recalculate. |
| Click "Reset to Defaults" | All inputs reset to defaults (Ramp: 10, Draw: 10, Removal: 8, Board Wipes: 3, Lands: 36). Must still click "Save Thresholds" to persist. |

### Loading & Degradation

| State | Behaviour |
|-------|-----------|
| Intelligence loading | Skeleton shimmer placeholders in synergy section. "SALT: LOADING..." text. No combo badges yet. Gap detection works immediately (local only). |
| EDHREC fails, Spellbook succeeds | Synergy shows fallback message. Salt shows N/A. Combos work normally. Gap detection works (uses local heuristics). |
| Both APIs fail | All intelligence sections show fallback messages. Deck builder remains fully functional. Single warning toast: "External intelligence unavailable." |
| Offline (cached data exists) | Intelligence sections populate from Dexie cache. No network requests. Stale indicator not shown (cache is valid for 7 days). |
| Offline (no cache) | Same as both APIs fail. |

## Responsive Breakpoints

Phase 4 components live entirely within the analytics sidebar (right panel of the three-panel deck builder layout). No new responsive behaviour required. Sidebar width is fixed by the existing three-panel layout from Phase 3.

Combo popover: max-width 320px, anchored to badge position. If badge is near viewport edge, popover shifts left to stay within viewport (CSS `right: 0` fallback).
