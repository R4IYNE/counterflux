# Phase 2 — UI Design Contract

> Visual and interaction contract for the Collection Manager (Treasure Cruise) screen. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (vanilla JS + Alpine.js, no component library) |
| Preset | not applicable |
| Component library | none (custom components, Organic Brutalism system) |
| Icon library | Material Symbols Outlined (font, `FILL` 0, `wght` 400, `opsz` 24) |
| Specialty icon fonts | mana-font (mana symbols), keyrune (set symbols) |
| Display font | Syne (self-hosted .woff2, 400/700) |
| Body font | Space Grotesk (self-hosted .woff2, 400/700) |
| Mono font | JetBrains Mono (self-hosted .woff2, 400/700) |
| CSS framework | Tailwind CSS v4 (CSS-first config via `@theme`, no `tailwind.config.js`) |
| Border radius | 0px everywhere (Organic Brutalism, no exceptions) |
| Charts | Chart.js 4.x (tree-shaken: bar, doughnut, pie) |

**Inherited:** Full design system carries forward from Phase 1 UI-SPEC. No new fonts, no new colour tokens. This phase adds components and interaction patterns within the established system.

**Source:** Phase 1 UI-SPEC, CONTEXT.md, DESIGN.md

### Label Weight Guidance (Phase 2 Specific)

| Context | Weight | Example |
|---------|--------|---------|
| Data values, table cell text, card prices, set names, timestamps | 400 | `EUR 48.20`, `MODERN HORIZONS 2`, `4,210 CARDS` |
| Overline labels, button text, column headers, filter labels, view mode labels | 700 | `ARCHIVE MANIFEST`, `EXPORT CSV`, `GALLERY`, `TABLE`, `SETS`, `FILTER:` |

### Body Weight Guidance (Phase 2 Specific)

Card tile names and set completion row names use the `body` token (Space Grotesk 14px) at weight 700 for emphasis. This avoids introducing a 5th font size while providing visual hierarchy within compact card and list layouts.

| Context | Font | Size | Weight | Example |
|---------|------|------|--------|---------|
| Card tile name (gallery) | Space Grotesk | 14px | 700 | `Lightning Bolt` (truncated with ellipsis) |
| Set name in completion view | Space Grotesk | 14px | 700 | `Modern Horizons 2` |
| Body prose, help text, descriptions | Space Grotesk | 14px | 400 | `Mila here! Your collection is empty...` |

## Interaction Patterns (Phase 2 Scope)

### View Switching

| Interaction | Behaviour |
|-------------|-----------|
| Click view toggle | Instant swap of content area (gallery/table/sets). No transition animation. Active toggle updates immediately. Filter state persists across view switches. Sort state persists. Scroll position resets to top. |

### Gallery View

| Interaction | Behaviour |
|-------------|-----------|
| Hover card tile | Border appears at `primary/50`. Image scales to 110% over 500ms. Image opacity goes from 80% to 100%. |
| Click card tile | Opens card detail flyout (Phase 1 component, now with "Add to Collection" functional). |
| Right-click card tile | Opens context menu with collection actions. |
| Scroll past 1,000 items | Virtual scroller activates. Items outside viewport are recycled. Smooth scrolling maintained. |

### Table View

| Interaction | Behaviour |
|-------------|-----------|
| Click column header | Sort by that column. First click: ascending. Second click: descending. Third click: remove sort. Active column header text turns primary. Sort direction icon (material symbol `arrow_upward`/`arrow_downward`) appears. |
| Hover row | Background shifts to surface-hover. |
| Click row | Opens card detail flyout. |
| Right-click row | Opens context menu. |

### Set Completion View

| Interaction | Behaviour |
|-------------|-----------|
| Click rarity filter | Recalculate progress bars for selected rarity tier only. Active filter: primary background. |
| Hover set row | Background shifts to surface-hover. |
| Click set row | Filter gallery/table to that set and switch to gallery view. |

### Filter Bar

| Interaction | Behaviour |
|-------------|-----------|
| Change sort dropdown | Re-sort current view immediately. No loading state needed (local data). |
| Click colour pip | Toggle colour filter. Active: filled with MTG colour + ring. Inactive: surface-hover background. Multiple can be active (AND filter). |
| Change category filter | Filter to Owned, Wishlist, or All. |
| All filters | Apply in combination. Empty result: show inline "No cards match your filters" message with Mila. |

### Add Card

| Interaction | Behaviour |
|-------------|-----------|
| Click "Add Card" | Opens add-card modal. |
| Search in modal | Debounced autocomplete (150ms, reuses Phase 1 search). Show top 8 results. |
| Select card from results | Populate modal with card name, image preview, current market price. |
| Adjust quantity | Number input, min 1, max 999. Default 1. |
| Toggle foil | Checkbox. When checked, price display switches to `prices.eur_foil`. |
| Select category | Radio: Owned (default) or Wishlist. |
| Confirm | Close modal. Add to IndexedDB. Toast success. Gallery/table updates reactively. |
| Add duplicate | If card already in collection (same scryfall_id + foil status), increment quantity. Toast: `{name} quantity updated to {new_qty}.` |

### Mass Entry Terminal

| Interaction | Behaviour |
|-------------|-----------|
| Click "Mass Entry" | Opens mass entry panel/modal. |
| Type/paste batch text | Textarea accepts multi-line input. JetBrains Mono font for terminal feel. |
| Click "Parse Entries" | Parse each line. Query Dexie for card resolution. Display results list. Resolved: green check + card thumbnail + name + set. Unresolved: red flag + original text + dropdown to pick from similar cards. |
| Fix unresolved entry | User selects correct printing from dropdown. Entry status changes to resolved. |
| Click "Commit to Collection" | All resolved entries added to IndexedDB in batch. Toast success with count. Unresolved entries remain in textarea for re-editing. |
| Click "Discard Entries" | Close panel. Confirmation if entries exist: "Discard {count} unparsed entries?" |

### CSV Import

| Interaction | Behaviour |
|-------------|-----------|
| Click "Import CSV" | Opens import modal. |
| Select file | Browser file picker. Accept `.csv` only. |
| Auto-detect format | Read first row headers. Match against known Deckbox/Moxfield/Archidekt column patterns. Display detected format. |
| Generic fallback | If no format detected, show column mapping dropdowns. User maps CSV columns to: Name (required), Quantity (required), Set (optional), Foil (optional). |
| Preview | Show first 10 rows in mini table. Display total count. |
| Click Import | Process all rows. Resolve card names against Dexie. Progress indicator. On completion: toast with count. Unresolved rows reported in a summary. |

### CSV Export

| Interaction | Behaviour |
|-------------|-----------|
| Click "Export CSV" | Immediately generate CSV and trigger browser download. Columns: Name, Set, Set Code, Collector Number, Quantity, Foil, Price EUR, Category. Toast success. |

### Analytics Panel

| Interaction | Behaviour |
|-------------|-----------|
| Click "Show Analytics" | Panel expands below stats header. Charts render with Chart.js. |
| Click "Hide Analytics" | Panel collapses. Charts destroyed (memory). |
| Hover chart segment | Tooltip shows label + count + percentage. |
| Click chart segment (colour/rarity) | Filter gallery/table to that segment. |

### Card Detail Flyout (Phase 2 Enhancements)

| Interaction | Behaviour |
|-------------|-----------|
| "Add to Collection" button | Now functional. Opens inline add form within flyout (quantity, foil, category). Confirm adds to collection. |

| Edit collection entry | If card is already in collection, show current qty/foil/category with inline edit controls. |

### Context Menu

|-------------|-----------|
| Right-click card | Menu appears at cursor position. 6 items max. Surface-hover background, ghost border, ambient shadow. Items: JetBrains Mono 11px 400 uppercase. Hover: primary text colour. |
