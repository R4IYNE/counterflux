<objective>
Add a hover-revealed checkbox affordance to every card tile in the Treasure Cruise gallery view that opens the same quick-action menu currently exposed only via right-click. Plan 3 of 3.

Purpose:
- **FOLLOWUP-3:** During the Phase 8 human-UAT walkthrough, the user reported that the only way to discover collection-edit actions (EDIT QUANTITY, TOGGLE FOIL, MOVE TO WISHLIST, REMOVE, etc.) is via right-click context menu — invisible to users who don't try right-clicking. Solution per the user: a hover-revealed checkbox per tile that opens the same menu. Critically: must be keyboard-accessible (tabbing into a tile + pressing Enter/Space must open the menu).

The existing context menu (`src/components/context-menu.js`) already dispatches via the `card-context-menu` custom event with `{ entry, x, y }` payload. This plan reuses that wiring — no new menu, no new actions, just a new opener.

Output: Updated `src/components/card-tile.js` (checkbox + keyboard handler), `src/components/gallery-view.js` (mirror change in virtual-scroller renderItem string template), `src/styles/main.css` (.card-quick-actions-checkbox utility), new `tests/gallery-view.checkbox.test.js` (5 assertions).

    // so we attach a single delegated listener on the scroller container.
    const scrollerEl = this.$el;
    const dispatchMenuFor = (tileEl, x, y) => {
      const entryId = tileEl.getAttribute('data-entry-id');
      if (!entryId) return;
      // entry.id is a string UUID; collection.entries lookup by id

    this._delegatedClick = (ev) => {
      const checkbox = ev.target.closest('.card-quick-actions-checkbox');
      if (!checkbox) return;
