Existing dropdown DOM (src/components/add-card-panel.js, lines 200-225):
```html
<div x-show="searchResults.length > 0"
     style="position: absolute; top: 100%; left: 0; right: 0; margin-top: 4px;
            background: var(--color-surface); border: 1px solid var(--color-border-ghost);
            max-height: 280px; overflow-y: auto; z-index: 10;">
  <template x-for="(card, idx) in searchResults" :key="card.id">
    <button ... 56px row ...>...</button>
  </template>
</div>
```
Container is wrapped in `<div style="position: relative;">` (line 188) inside the
`<aside class="tc-panel-column">` whose own style declares `overflow-y: auto`.

THE BUG: the parent `<aside>` has `overflow-y: auto`. When an absolutely-positioned
child overflows the aside's content box, the aside's overflow:auto clips it instead
of letting it visually escape. The dropdown's own max-height/overflow-y is correct
in isolation but is overridden by the parent's clipping.

Existing re-open affordance (src/screens/treasure-cruise.js, lines 42-52):
```html
<button

  @click="$store.collection.togglePanel()"

         display: inline-flex; align-items: center; justify-content: center;
         background: var(--color-surface); border: 1px solid var(--color-border-ghost);
         color: var(--color-text-muted); cursor: pointer; z-index: 10;">
  <span class="material-symbols-outlined" style="font-size: 20px;">chevron_right</span>
</button>
```
The user found this too subtle. Per follow-ups.md: brighter (primary accent vs ghost),
