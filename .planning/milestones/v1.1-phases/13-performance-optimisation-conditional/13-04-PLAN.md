<!-- Visual-identity constraints (CLAUDE.md) — layout reservations MUST respect these -->

8-point spacing scale (mandatory, no exceptions):
- 4px  (gap-xs)
- 8px  (gap-sm)
- 16px (gap-md)
- 24px (gap-lg)
- 32px (gap-xl)
- 48px (gap-2xl)
- 64px (gap-3xl)

Any `min-height: 27px` or `aspect-ratio: 0.713` is a token violation. Use 28/32/48/etc.

<!-- Mana font / keyrune / card-image aspect ratios (standard MTG card) -->

MTG card aspect ratio: 63:88 (≈ 0.716), normal Scryfall large: 488×680.
For card tiles, use `aspect-ratio: 63 / 88;` or pixel equivalents that land on the 8-point grid.
</interfaces>

<audit_protocol>
<!-- Per Research §Pattern 5 applied to CLS — audit-driven fix workflow -->

1. After Plan 3 merges, run a fresh Lighthouse on http://localhost:4173 (desktop preset, Performance only). Capture the CLS number + the "Avoid layout shifts" insight from `lighthouse-report/report.html`.
2. The insight lists elements with their shift amounts ("Element [div.foo] shifted by 0.23"). Take the top 3 by shift amount.
3. For each of the top 3, inspect the element and classify the root cause:
   - **Image without dimensions:** add `width`/`height` or `aspect-ratio`
   - **Dynamic content insertion:** reserve space with `min-height` matching the eventual content
   - **Font swap:** add `font-display: optional` or preload the font
   - **Animated layout (width/height transition):** convert to transform (Plan 2 Task 3 already closes this for layout-triggering animations — cross-check)
   - **User-triggered expansion:** genuinely unavoidable — document in 13-CLS-DELTA.md as accepted shift
4. Apply the fix, re-measure CLS via `npm run perf`, record before/after in 13-CLS-DELTA.md.
5. Repeat for the next contributor if CLS still > 0.1.
</audit_protocol>
