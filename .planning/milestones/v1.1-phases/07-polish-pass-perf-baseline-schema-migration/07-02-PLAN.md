<baseline_methodology>
<!-- D-18 + D-22a — what "median of 3 runs" means concretely -->
1. Build production bundle: `npm run build`
2. Boot preview server: `npm run preview` (defaults to http://localhost:4173)
3. Open Chrome → DevTools → Lighthouse panel → desktop preset → "Performance" only → Analyze
4. Record LCP, TTI, FCP, INP, CLS, TBT (Total Blocking Time)
5. Repeat steps 3-4 two more times. Each run takes ~30s.
6. Compute median across the 3 runs for each metric (use the middle value).
7. Set targets per Web Vitals "Good" thresholds:
   - LCP: < 2.5s
   - INP: < 200ms
   - CLS: < 0.1
   - TTI: ≤ measured median + 0% (or aspirational < 3.5s if measured baseline already meets)
8. Write to PERF-BASELINE.md.

If baseline is already inside the "Good" thresholds, set targets at the measured median. If baseline is in "Needs Improvement", set targets at the "Good" threshold and flag for Phase 13.
</baseline_methodology>
