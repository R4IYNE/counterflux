## Verification

Both fixes are additive. The sets sort applies to cache writes too, so the next 24h of sessions inherit the ordering even before a fresh fetch. Bulk-action UI hidden when there's only 1 row to keep the simple case unchanged.

Sequential bulk dispatch (rather than `Promise.all`) is deliberate: Phase 11's sync engine push respects FK ordering via `PUSH_ORDER` and does its own batching, but firing 848 parallel `store.retry()` calls would let races interleave the queue+conflicts state in ways the existing tests don't cover. Sequential keeps the user-visible per-row state observable and matches the cadence Supabase expects.
