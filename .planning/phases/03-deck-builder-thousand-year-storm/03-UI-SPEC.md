# Phase 3 — UI Design Contract

> Visual and interaction contract for the Deck Builder (Thousand-Year Storm) phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (vanilla JS + Alpine.js, no component library) |
| Preset | not applicable |
| Component library | none (custom components, Organic Brutalism system) |
| Icon library | Material Symbols Outlined (font, `FILL` 0, `wght` 400, `opsz` 24) |
| Specialty icon fonts | mana-font (mana symbols), keyrune (set symbols) |
| Display font | Syne (self-hosted .woff2, 400/700) |
| Body font | Space Grotesk (self-hosted .woff2, 400/700) |
| Mono font | JetBrains Mono (self-hosted .woff2, 400/700) |
| Serif accent | Crimson Pro (self-hosted .woff2, 400/700) -- reserved for flavour text quotes |
| CSS framework | Tailwind CSS v4 (CSS-first config via `@theme`, no `tailwind.config.js`) |
| Border radius | 0px everywhere (Organic Brutalism, no exceptions) |
| Drag library | SortableJS 1.15.x |

**Source:** Phase 1 UI-SPEC design system (inherited). SortableJS from CONTEXT.md code context and REQUIREMENTS.md DECK-06.

### Label Weight Guidance (Phase 3 contexts)

| Context | Weight | Example |
|---------|--------|---------|
| Stat values, metadata, tag text, prices, timestamps | 400 | "LAST EDITED 2 DAYS AGO", tag pill text, price values |
| Section headers, overline labels, button text, category headers, card count | 700 | "01 // CREATURES (23)", "INITIALIZE RITUAL", "ARCHIVE ANALYTICS", "MANA CURVE" |

### Additional Rule for Phase 3

- Category type headers use `label` tier at 700 weight with format: `TYPE_NAME (COUNT)` -- e.g., `CREATURES (23)`, `INSTANTS (8)`
- The `67/99` card count uses `display` size for the numbers, with "slots remaining" as `label` 400 beneath it
- Deck card names in the landing grid use `label` 700
- Format badges (COMMANDER, STANDARD, etc.) use `label` 400 with `border-ghost` border

**Source:** Phase 1 UI-SPEC typography (inherited). CONTEXT.md D-18 for card count display.

## Interaction Patterns (Phase 3 Scope)

### Deck Landing

| Interaction | Behaviour |
|-------------|-----------|
| Click deck card | Open three-panel editor for that deck |
| Right-click deck card | Show deck landing context menu |
| Click "Initialize Ritual" | Open ritual modal wizard |
| Empty state | Mila illustration + "No Decks in the Archive" copy + "Initialize Ritual" button |

### Initialize Ritual Modal

| Interaction | Behaviour |
|-------------|-----------|
| Open | Glass overlay backdrop. Modal centred. Focus trapped inside modal. |
| Commander search | Autocomplete filtered to legendary creatures only. 150ms debounce. |
| Partner detected | If selected commander has Partner keyword, show second search field. Colour identity auto-merges. |
| Companion slot | Optional field, always visible but not required. |
| Deck name | Auto-fills from commander name. User can edit freely. |
| Format dropdown | Default: Commander (100). Other options: Standard (60), Modern (60), Legacy (60), Vintage (60), Pauper (60). |
| "Begin Ritual" click | Validate commander selected. Create deck in Dexie. Navigate to three-panel editor. Toast: "Deck created." |
| Cancel / Escape / backdrop click | Close modal, discard inputs. |

### Three-Panel Editor

| Interaction | Behaviour |
|-------------|-----------|
| Panel resize | Fixed widths: search 280px, analytics 280px, centre fills remaining. Not user-resizable. |
| Back navigation | Click deck name breadcrumb or browser back returns to deck landing. |

### Search Panel

| Interaction | Behaviour |
|-------------|-----------|
| Type in search | 150ms debounce. Query filtered by commander colour identity. Results populate below. |
| "In Collection Only" toggle | ON: filter results to cards in `collection` store. OFF: show all cards (default). Toggle styled as primary bg when active, ghost border when inactive. |
| Click search result | Add card to deck. Toast: "{name} added." Card count increments. Analytics recalculate (<100ms). |
| Drag search result | SortableJS drag from search results to centre panel. Drop zone highlights with primary border + glow-blue. |
| Right-click search result | Context menu: Add to Deck, Add to Collection, Add to Wishlist, View Details, View on Scryfall. |
| Duplicate add (Commander) | Toast warning: "{name} is already in this deck. Commander format enforces singleton." Card NOT added. |
| Duplicate add (60-card) | Increment quantity. Toast: "{name} x{qty}." |

### Centre Panel

| Interaction | Behaviour |
|-------------|-----------|
| View toggle: Grid | Card image tiles in groups by type. SortableJS drag-and-drop within and between groups. |
| View toggle: List | Compact table rows (name, type, mana, price, tags). No drag-and-drop in list view. |
| Drag card between groups | SortableJS handles. Drop zone: 2px primary border on target group, glow-blue shadow. Ghost element follows cursor at 60% opacity. |
| Right-click card | Deck context menu (see copywriting contract). |
| Click card | Open card detail flyout (reuse Phase 1/2 flyout). |
| Category headers | Collapsible. Click header to expand/collapse group. Chevron icon rotates. |
| Tag pills on cards | Small pills below card image (in grid) or in tag column (in list). Clicking a tag in filter bar highlights cards with that tag across all groups. |

### Analytics Panel

| Interaction | Behaviour |
|-------------|-----------|
| Chart updates | All charts recalculate within 100ms on any card add/remove/move. Use `requestAnimationFrame` for batching. |
| Mana curve hover | Tooltip shows exact count for hovered CMC bar. |
| Colour pie hover | Tooltip shows colour name + count + percentage. |
| Salt score section | Non-interactive placeholder. Text-dim label: "COMING IN PHASE 4". |

### Drag-and-Drop (SortableJS)

| Interaction | Behaviour |
|-------------|-----------|
| Drag start | Card tile lifts: `surface-hover` background, 4px ambient shadow, 60% opacity ghost. Cursor changes to `grabbing`. |
| Drag over valid drop zone | Target group header gets 2px primary left border. Group area gets subtle `glow-blue` box-shadow. |
| Drag over invalid zone | No highlight. Drop reverts to origin. |
| Drop | Card moves to target group. Card count unchanged (reorder only). Animation: 150ms ease slide-in. |
| Search-to-deck drag | Dropping from search panel into centre panel adds card to deck in the target group. Card count increments. |
| Handle | Entire card tile is the drag handle (no separate handle icon). Cursor: `grab` on hover. |

### Import/Export

| Interaction | Behaviour |
|-------------|-----------|
| Import button click | Open import modal. |
| Paste decklist | Auto-detect format. Show format badge. Parse and resolve cards. |
| File drop | Accept `.txt`, `.dec`, `.csv` files. Parse content. |
| Unresolved cards | Show flagged list with manual matching interface (reuse Phase 2 mass entry resolution pattern). |
| Export button click | Open export modal with format selection. |
| Copy to clipboard | Copy formatted decklist. Toast: "Decklist copied to clipboard." |
| Download | Download file with appropriate extension (.txt, .dec, .csv). |

### Context Menu

| Interaction | Behaviour |
|-------------|-----------|
| Open | Right-click dispatches custom DOM event (`deck-context-menu` or `deck-landing-context-menu`). Menu renders at cursor position. |
| Click menu item | Execute action. Menu closes. |
| Click outside / Escape | Menu closes. |
| Destructive items | Text colour: secondary (#E23838). |
| "View on Scryfall" | Opens `scryfall.com/card/{set}/{collector_number}` in new tab. |

### Panel Separation

Panels are separated by **tonal shifting** (No-Line Rule), not borders:
- Search panel: `surface` (#14161C) background
- Centre panel: `background` (#0B0C10) background
- Analytics panel: `surface` (#14161C) background

This creates the visual rhythm: dark-light-dark across the three panels.

### Deck Landing Grid

- Grid of deck cards: `grid-cols-3` on wide viewports (>= 1280px), `grid-cols-2` below
- Each deck card: `surface` background, `surface-hover` on hover
- Commander art thumbnail: Scryfall `art_crop` image, 16:9 aspect ratio, top of card
- Card content below: deck name (label 700), card count, format badge, last edited
