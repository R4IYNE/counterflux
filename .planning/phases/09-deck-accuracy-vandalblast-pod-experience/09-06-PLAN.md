Fix the two cascading gaps from Phase 9 HUMAN-UAT: (6) turn timer doesn't auto-start on Start Game or NEXT TURN — user has to manually click the play button in the floating toolbar, and (7) TURN PACING section doesn't render on the post-game overlay (cascades from gap 6 — if the timer never runs, `turnStartedAt` stays at the initial anchor from `startGame()`, so laps DO accumulate via `Date.now() - turnStartedAt` even without `startTimer()` running; but if the user never clicks NEXT TURN, no laps are pushed; and users report the TURN PACING section is missing entirely).

Purpose: The architectural bug is that `startTimer()` is currently ONLY called from the manual play/pause button in the floating toolbar (`floating-toolbar.js` line 59). Per GAME-09 + GAME-10 spec, the timer should auto-start when a game begins and reset on each NEXT TURN. Fixing this resolves gap 6 directly AND gap 7 indirectly (verified by a new end-to-end test that simulates a 3-turn game).

Output: Timer auto-starts on startGame() and nextTurn(). turn_laps accumulates reliably. TURN PACING renders with real data post-game.

startTimer() {
  if (this.timerRunning) return;
  this.timerRunning = true;
  if (this.turnStartedAt == null) this.turnStartedAt = Date.now();
  const tick = () => {
    if (!this.timerRunning) return;
    this.timerSeconds = Math.floor((Date.now() - this.turnStartedAt) / 1000);
    this._timerRafId = requestAnimationFrame(tick);
  };
  this._timerRafId = requestAnimationFrame(tick);
}
