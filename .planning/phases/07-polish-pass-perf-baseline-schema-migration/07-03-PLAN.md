<temp_table_shuffle_canonical_pattern>
<!-- D-01a: v6 + v7 in the same file. Both bumps ship in this PR. -->

**v6 — shadow tables phase (variant a, chosen by Task 3a spike):**
- Keeps the OLD `collection`, `decks`, `deck_cards`, `games`, `watchlist` tables in v6 declaration (so .upgrade() can read from them)
- Adds NEW shadow tables with a `_next` suffix (`collection_next`, `decks_next`, `deck_cards_next`, `games_next`, `watchlist_next`) with the target UUID PK + new sync fields
- Adds NEW sync_queue, sync_conflicts, profile tables under their FINAL clean names (no rename ever needed for these)
- Adds price_history.updated_at column (no PK change — straight modify via `.modify()`)
- Adds schema_version row to meta

**v7 — drop-legacy phase:**
- Drops old autoincrement tables by declaring them as `null` per Dexie idiom: `collection: null, decks: null, deck_cards: null, games: null, watchlist: null`
- Keeps `*_next` tables declared with the same shape as v6 (they carry the migrated data forward)
- Does NOT attempt to rename `*_next` back to clean names in the same literal — JS duplicate-key rules prevent that. `*_next` becomes the PERMANENT name consumed by Phases 9 and 11.

**Why two versions:** Dexie 4.x cannot change a table's PK type in one version AND cannot rename a table. Pattern verified via Dexie issues #1148 and #646 and confirmed by Task 3a spike. Both v6 and v7 ship in the same Plan 3 PR per D-01a/D-24.

**FK ordering rule (Pitfall C):** Inside v6.upgrade, build deckRemap FULLY (decks_next fully populated) BEFORE iterating deck_cards. Strict sequential — no Promise.all parallelism on remap-dependent loops.

**Progress emission (D-17a):** Inside the upgrade callback, count total rows across all six synced tables FIRST, then emit a progress event every 10% of progress. Receiver = `Alpine.store('bulkdata').migrationProgress`. Splash-screen (Plan 1 Task 1) renders the value.
</temp_table_shuffle_canonical_pattern>

<backfill_values_per_decision>
- updated_at: Date.now() at migration time (D-07) — same value for every row in this batch
- synced_at: null (D-08)
- turn_laps: existing value if present, else [] (D-09 — `game.turn_laps || []`)
- user_id: null on every synced row (Phase 10 sets it on auth)
- price_history.updated_at: Date.now() if not already set (D-11 — uses .modify(), no PK change)
- meta.schema_version row: { key: 'schema_version', version: 7, migrated_at: new Date().toISOString() } (D-12)
</backfill_values_per_decision>

       ```javascript
       /**
        * Pre-migration backup service (SCHEMA-03).
        * Per D-13: backs up collection, decks, deck_cards, games, watchlist, price_history.
        * Per D-14: single localStorage key `counterflux_v5_backup_<ISO-timestamp>`, synchronous write.
        * Per D-16: 7-day TTL sweep on every boot.
        * Per D-17b: writes then reads back + JSON.parses to validate round-trip integrity.
        * Per Pitfall D: on QuotaExceededError, falls back to file-download.

       export const BACKUP_KEY_PREFIX = 'counterflux_v5_backup_';
       export const BACKUP_TTL_MS = 7 * 24 * 60 * 60 * 1000; // D-16
       export const USER_TABLES = ['collection', 'decks', 'deck_cards', 'games', 'watchlist', 'price_history']; // D-13

       export class MigrationBackupFailedError extends Error {
         constructor(message, cause) { super(message); this.name = 'MigrationBackupFailedError'; this.cause = cause; }
       }

       export async function backupBeforeMigration(db) {
         // Skip if fresh install OR already migrated (D-13 implicit)
         const currentVersion = db.verno;
         if (currentVersion === 0 || currentVersion >= 6) {
           console.info('[migration-backup] skipping — db.verno=', currentVersion);
           return null;
         }

         // 1. Snapshot every user table (D-13 scope)
         const snapshot = {};
         let totalRows = 0;
         for (const table of USER_TABLES) {
           try {
             snapshot[table] = await db.table(table).toArray();
             totalRows += snapshot[table].length;
           } catch (e) {
             // Table may not exist on older versions (e.g., watchlist is v5+, games is v5+)
             snapshot[table] = [];
           }
         }

         const isoTs = new Date().toISOString();
         const key = `${BACKUP_KEY_PREFIX}${isoTs}`;
         const json = JSON.stringify(snapshot);

         // 2. Write — handle quota (Pitfall D)
         try {
           localStorage.setItem(key, json);
         } catch (e) {
           if (e?.name === 'QuotaExceededError') {
             try {
               downloadBackupFile(json, isoTs);
             } catch {/* swallow — primary error is more useful */}
             throw new MigrationBackupFailedError(
               `localStorage quota exceeded (${(json.length / 1024 / 1024).toFixed(2)}MB). A downloadable backup was attempted — check Downloads.`,
               e,
             );
           }
           throw new MigrationBackupFailedError('Backup write failed', e);
         }

         // 3. Round-trip validate (D-17b)
         const readback = localStorage.getItem(key);
         if (!readback) {
           throw new MigrationBackupFailedError('Backup write succeeded but readback returned null');
         }
         let parsed;
         try {
           parsed = JSON.parse(readback);
         } catch (e) {
           localStorage.removeItem(key); // don't leave a corrupt backup
           throw new MigrationBackupFailedError('Backup written but JSON.parse failed on read-back', e);
         }
         let parsedRowCount = 0;
         for (const table of USER_TABLES) {
           parsedRowCount += (parsed[table] || []).length;
         }
         if (parsedRowCount !== totalRows) {
           localStorage.removeItem(key);
           throw new MigrationBackupFailedError(
             `Backup row count mismatch: wrote ${totalRows}, read ${parsedRowCount}`,
           );
         }

         console.info(`[migration-backup] saved & validated (${(json.length / 1024).toFixed(1)}KB, ${totalRows} rows) at key=${key}`);
         return key;
       }

       export function sweepOldBackups(now = Date.now()) {
         const removed = [];
         // Iterate backwards because we mutate
         for (let i = localStorage.length - 1; i >= 0; i--) {
           const key = localStorage.key(i);
           if (!key?.startsWith(BACKUP_KEY_PREFIX)) continue;
           const isoTs = key.substring(BACKUP_KEY_PREFIX.length);
           const ts = new Date(isoTs).getTime();
           if (Number.isNaN(ts)) continue;
           if (now - ts > BACKUP_TTL_MS) {
             localStorage.removeItem(key);
             removed.push(key);
           }
         }
         if (removed.length) {
           console.info(`[migration-backup] swept ${removed.length} old backup(s):`, removed);
         }
         return removed;
       }

       export function listBackups() {
         const out = [];
         for (let i = 0; i < localStorage.length; i++) {
           const key = localStorage.key(i);
           if (key?.startsWith(BACKUP_KEY_PREFIX)) out.push(key);
         }
         return out;
       }

       export function restoreFromBackup(key) {
         const json = localStorage.getItem(key);
         if (!json) throw new MigrationBackupFailedError(`Backup key not found: ${key}`);
         return JSON.parse(json);
       }

       function downloadBackupFile(json, isoTs) {
         const blob = new Blob([json], { type: 'application/json' });
         const url = URL.createObjectURL(blob);
         const a = document.createElement('a');
         a.href = url;
         a.download = `counterflux-v5-backup-${isoTs}.json`;
         document.body.appendChild(a);
         a.click();
         document.body.removeChild(a);
         URL.revokeObjectURL(url);
       }
       ```

       ```javascript
             synced_at: null,
           });
           processed++; emitProgress();
         }

         // 2. decks → decks_next (Pitfall C: FULLY populate deckRemap BEFORE deck_cards loop)
         const oldDecks = await tx.table('decks').toArray();
         const deckRemap = new Map();
         for (const deck of oldDecks) {
           const newId = crypto.randomUUID();
           deckRemap.set(deck.id, newId);
           await tx.table('decks_next').add({
             ...deck,
             id: newId,
             user_id: null,
             updated_at: deck.updated_at || now,
             synced_at: null,
           });
           processed++; emitProgress();
         }
         if (deckRemap.size !== oldDecks.length) {
           throw new Error(`[migration v6] deckRemap size mismatch: ${deckRemap.size} vs ${oldDecks.length}`);
         }

         // 3. deck_cards → deck_cards_next with FK remap (D-02). Orphans skipped (Pitfall C).
         const oldDeckCards = await tx.table('deck_cards').toArray();
         for (const dc of oldDeckCards) {
           const newDeckId = deckRemap.get(dc.deck_id);
           if (!newDeckId) {
             console.warn('[migration v6] orphan deck_card, skipping:', dc);
             processed++; emitProgress();
             continue;
           }
           await tx.table('deck_cards_next').add({
             ...dc,
             id: crypto.randomUUID(),
             deck_id: newDeckId,
             user_id: null,
             updated_at: now,
             synced_at: null,
           });
           processed++; emitProgress();
         }

         // 4. games → games_next (+ turn_laps backfill D-09 + deck_id remap)
         const oldGames = await tx.table('games').toArray();
         for (const game of oldGames) {
           const newDeckId = game.deck_id != null ? (deckRemap.get(game.deck_id) ?? null) : null;
           await tx.table('games_next').add({
             ...game,
             id: crypto.randomUUID(),
             deck_id: newDeckId,
             user_id: null,
             turn_laps: Array.isArray(game.turn_laps) ? game.turn_laps : [],
             updated_at: now,
             synced_at: null,
           });
           processed++; emitProgress();
         }

         // 5. watchlist → watchlist_next
         const oldWatchlist = await tx.table('watchlist').toArray();
         for (const row of oldWatchlist) {
           await tx.table('watchlist_next').add({
             ...row,
             id: crypto.randomUUID(),
             user_id: null,
             updated_at: now,
             synced_at: null,
           });
           processed++; emitProgress();
         }

         // 6. price_history.updated_at backfill (D-11, no PK change)
         await tx.table('price_history').toCollection().modify(row => {
           if (row.updated_at == null) row.updated_at = now;
         });

         // 7. schema_version meta row (D-12)
         await tx.table('meta').put({
           key: 'schema_version',
           version: 6,
           migrated_at: new Date().toISOString(),
         });

         emitProgress(); // final 100%
       });

       // ============================================================
       // Schema v7 — rename phase (variant a)
       // Drops the legacy autoincrement tables (`collection: null` etc.).
       // The *_next tables remain populated; v7.upgrade() copies their rows into
       // freshly recreated final-name tables (same keys, new shape) and clears *_next.
       // Dexie 4.x handles this because each literal key appears at most once:
       //   v7 drops the OLD `collection` (different shape from v6 → null-drop),
       //   then the v7 upgrade populates fresh `collection` rows from collection_next.
       // The spike (Task 3a) confirms Dexie tolerates the null-then-recreate sequence ACROSS versions.
       // ============================================================
       db.version(7).stores({
         // Drop the old autoincrement tables (key appears once: as null)
         collection: null,
         decks: null,
         deck_cards: null,
         games: null,
         watchlist: null,
         // Keep *_next populated for one more version so v7.upgrade can copy from them.
         // They will be nulled in a future v8 once we're confident every user has reached v7.
         collection_next: 'id, scryfall_id, category, foil, user_id, updated_at, synced_at, [scryfall_id+foil], [scryfall_id+category]',
         decks_next: 'id, name, format, user_id, updated_at, synced_at',
         deck_cards_next: 'id, deck_id, scryfall_id, user_id, updated_at, synced_at, [deck_id+scryfall_id]',
         games_next: 'id, deck_id, user_id, started_at, ended_at, updated_at, synced_at',
         watchlist_next: 'id, &scryfall_id, user_id, updated_at, synced_at',
         // Unchanged
         cards: 'id, name, oracle_id, set, collector_number, cmc, color_identity, type_line, [set+collector_number]',
         meta: 'key',
         price_history: '++id, scryfall_id, date, updated_at, [scryfall_id+date]',
         edhrec_cache: 'commander',
         combo_cache: 'deck_id',
         card_salt_cache: 'sanitized',
         profile: 'id, user_id, updated_at',
         sync_queue: '++id, table_name, user_id, created_at',
         sync_conflicts: '++id, table_name, detected_at',
       }).upgrade(async tx => {
         // The legacy collection/decks/deck_cards/games/watchlist tables were dropped by the stores()
         // declaration above. We need to create them anew with the final UUID-PK shape. But v7's
         // stores({}) nulled them — Dexie cannot declare a key twice in one literal.
         //
         // Solution: ship a follow-up version bump (v8) that RECREATES the clean names. OR do the
         // copy INSIDE this upgrade by opening the raw IDB transaction and creating object stores
         // directly. The cleaner path is to DEFER the rename — keep *_next as the live table names
         // through v7, and do the rename in v8 of a future phase.
         //

         // Phases 9 and 11 to consume. Downstream consumers read `db.collection_next`. The legacy

         //
         // Update schema_version meta row.
         await tx.table('meta').put({
           key: 'schema_version',
           version: 7,
           migrated_at: new Date().toISOString(),
         });
       });
       ```

       ```javascript
        * Migration orchestrator (SCHEMA-01..03).
        * Runs BEFORE Alpine and any other store init. Gates app boot.
        * - Sweeps old backups (D-16)
        * - Writes pre-migration backup with round-trip validation (D-17b)

       import { showBlockingModal, hideBlockingModal } from '../components/migration-blocked-modal.js';

       export async function runMigration() {
         // 1. Sweep stale backups (D-16) — best-effort
         try { sweepOldBackups(); } catch (e) { console.warn('[migration] sweep failed', e); }

         // 2. Register blocked handler BEFORE opening (Pitfall F)
         //    db.on() must be called BEFORE db.open(); Dexie attaches the listener early.
         db.on('blocked', (event) => {
           console.warn('[migration] blocked by another tab', event);
           showBlockingModal({
             title: 'Counterflux is upgrading',
             body: 'Please close any other Counterflux tabs so we can finish updating your local data.',
           });
         });
         db.on('ready', () => {
           hideBlockingModal();
         });
         db.on('versionchange', (event) => {
           console.warn('[migration] versionchange — another tab is upgrading; closing connection', event);
           db.close();
           // Show a non-blocking notice (toast); the page will need a manual reload
           // (We don't auto-reload because the user may have unsaved work — pure data app, but be safe)
           if (typeof window !== 'undefined' && window.alert) {
             // Plain alert is intentional — runs before Alpine + toast system are initialized
             setTimeout(() => alert('Counterflux was updated in another tab. Please reload this page.'), 0);
           }
         });

         //    Dexie won't tell us verno without opening, so we open at the FULL schema (chain ends at v7).
         //    The .upgrade callbacks in schema.js will run for v6 and v7 if currentVersion < 6.
         //    BUT: backupBeforeMigration needs to read v5 data BEFORE the upgrade runs.
         //    Solution: open a separate Dexie instance at v5 ONLY, snapshot, close, then open the real db.
         try {

           // Skip the probe if the DB is brand new (we detect this with indexedDB.databases() if available)
           const isExisting = await detectExistingDb('counterflux');

             // Probe at v5 to snapshot pre-upgrade data for the backup.
             // Post-migrated users (already at v6+) will throw VersionError here — that is EXPECTED
             // and means "no backup needed, skip silently and proceed to db.open()".
             let probeDb;
             try {
               probeDb = await openProbeAtV5();

               const name = probeErr?.name || probeErr?.inner?.name;
               if (name === 'VersionError') {

               }

                 await backupBeforeMigration(probeDb);
               } finally {
                 probeDb.close();
               }
             }
           }

           // 4. Open the real db — this triggers v6 and v7 upgrades if needed
           await db.open();

           console.info('[migration] complete; db.verno=', db.verno);
         } catch (e) {

           if (e instanceof MigrationBackupFailedError) {

             showBlockingModal({
               title: 'Backup failed — migration aborted',

               title: 'Migration failed',

             });

           try {

               const store = window.Alpine.store('bulkdata');
               if (store) store.migrationProgress = null;
             }
           } catch { /* swallow */ }
         }
       }

         if (typeof indexedDB.databases === 'function') {

         // Fallback: try opening at v1, check verno > 0
         return true; // safe default — backup is no-op for fresh DBs anyway

         await probe.open();
         return probe;
       }
       ```
