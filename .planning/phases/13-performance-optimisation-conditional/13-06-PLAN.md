# Plan 6 — Soft-gate @lhci/cli + Phase 13 Signoff

**Trigger:** Unconditional — always executes as Wave 4 closer.
**Depends on:** Plan 1 (for Branch decision + REMEASURE artefact).
**Soft dependency:** Plans 2/3/4/5 if they triggered (this plan enumerates them in signoff).

## Objective

Install a PR-time Lighthouse soft-gate so v1.2+ regressions surface in code review without blocking merges (D-11, D-12). Close Phase 13 with `13-PERF-SIGNOFF.md` containing the authoritative v1.1 perf verdict — either confirming Plan 1's Branch A signoff, or writing the Branch B signoff that enumerates every optimisation shipped and captures the final re-measured numbers.

<context>
### Why soft gate (not hard gate)

Phase 7 D-20 explicitly deferred this decision to Phase 13. CONTEXT.md D-11 locks soft gate: `@lhci/cli` 0.15.1 `level: 'warn'` assertions print to the workflow output but do not exit non-zero. For a solo-developer project, hard-blocking on perf regression is a poor tradeoff for hotfixes and emergency merges. Soft gate gives us visible regression signals without the friction.

Research §Pattern 3 documents the three-level grammar (`off | warn | error`) supported natively by @lhci/cli 0.15.1 — no wrapper script needed.

### Why warnings must surface

Research §Pitfall 4: silent warnings are ignored warnings. The GitHub workflow MUST post the warning summary somewhere visible — either PR comment (requires `GITHUB_TOKEN` + `pull_request` permission) or GITHUB_STEP_SUMMARY (appears on the Actions run page). We use GITHUB_STEP_SUMMARY as the primary because it doesn't require PR-write permission on forked PRs.

### Signoff artefact shape (D-13, D-14)

13-PERF-SIGNOFF.md is a NEW artefact in the phase directory. It cross-references PERF-BASELINE.md (Phase 7 frozen per D-14) as the v1.0 comparison anchor. Plan 1 created the file — either finalised on Branch A or stubbed on Branch B. This plan's final task resolves whichever state Plan 1 left it in.

</context>

  <name>Task 3: Write lhci-warn-summary.cjs helper for GITHUB_STEP_SUMMARY formatting</name>
  <files>scripts/lhci-warn-summary.cjs</files>
  <read_first>

  <action>
  Create `scripts/lhci-warn-summary.cjs`:

  ```js
  #!/usr/bin/env node
  /**
   * Phase 13 Plan 6 — @lhci/cli warn-level summary formatter.
   *
   * Reads the assertion results that `lhci assert` emits to `.lighthouseci/assertion-results.json`
   * after `lhci autorun`, formats a markdown table, writes to $GITHUB_STEP_SUMMARY so the
   * Actions UI renders it inline on the run page. Exits 0 always (soft-gate invariant).
   *
   * Runs AFTER `lhci autorun` in the workflow — per Research §Pitfall 4 (silent warnings
   * are ignored warnings).
   */
  const fs = require('node:fs');
  const path = require('node:path');

  const RESULTS_PATH = path.resolve(process.cwd(), '.lighthouseci/assertion-results.json');
  const SUMMARY_PATH = process.env.GITHUB_STEP_SUMMARY;

  const TARGETS = {
    'largest-contentful-paint': 'LCP',
    'cumulative-layout-shift': 'CLS',
    'first-contentful-paint': 'FCP',
    'interaction-to-next-paint': 'INP',
  };

  if (!fs.existsSync(RESULTS_PATH)) {
    console.log('[lhci-warn-summary] No assertion-results.json — nothing to report.');
    process.exit(0);
  }

  const results = JSON.parse(fs.readFileSync(RESULTS_PATH, 'utf-8'));
  const warnings = results.filter(r => r.level === 'warn' && !r.passed);

  const lines = ['## 🔦 Lighthouse soft-gate (Phase 13 Plan 6)', ''];
  if (warnings.length === 0) {
    lines.push('All Web Vitals within Phase 7 targets. ✅');
  } else {
    lines.push('| Metric | Actual | Target | Status |');
    lines.push('|--------|--------|--------|--------|');
    for (const w of warnings) {
      const metric = TARGETS[w.auditId] || w.auditId;
      const actual = typeof w.actual === 'number' ? w.actual.toFixed(w.auditId === 'cumulative-layout-shift' ? 3 : 0) : w.actual;
      const target = w.expected || '—';
      lines.push(`| ${metric} | ${actual} | ${target} | ⚠️ warn (soft) |`);
    }
    lines.push('');
    lines.push('**Soft gate — merge NOT blocked.** Investigate if regression is intentional. See `.planning/phases/13-performance-optimisation-conditional/13-PERF-SIGNOFF.md` for v1.1 targets.');
  }

  const output = lines.join('\n') + '\n';
  console.log(output);

  if (SUMMARY_PATH) {
    fs.appendFileSync(SUMMARY_PATH, output);
    console.log('[lhci-warn-summary] Wrote summary to $GITHUB_STEP_SUMMARY');
  } else {
    console.log('[lhci-warn-summary] No GITHUB_STEP_SUMMARY env — local run; stdout only.');
  }

  process.exit(0);
  ```

  Make the file executable (chmod +x on Unix; on Windows the `node scripts/lhci-warn-summary.cjs` invocation in the workflow bypasses the exec bit).
