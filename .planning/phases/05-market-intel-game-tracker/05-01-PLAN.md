**3. Price history service** (`src/services/price-history.js`):
Export these functions:
- `snapshotWatchlistPrices()` -- for each watchlist entry, check if today's snapshot exists via `[scryfall_id+date]` compound index. If not, read price from `cards` table, add to `price_history`. Prune records older than 90 days. Use `new Date().toISOString().slice(0, 10)` for date strings.
- `computeTrend(scryfallId, days)` -- query price_history for given card and period, return `{ prices: number[], change: number, changePct: number, direction: 'up'|'down'|'flat' }`. Prices are EUR values from snapshots. Change = latest - earliest.
- `computeMovers(period, limit = 10)` -- query all price_history for the period, group by scryfall_id, compute absolute EUR change, return `{ gainers: [{scryfall_id, change, changePct, currentPrice}], losers: [...] }` sorted by absolute change. Limit to top N each.

Import `db` from `../db/schema.js`.

**4. Sets service** (`src/services/sets.js`):
Export:
- `fetchSets()` -- check IndexedDB `meta` table for key `scryfall-sets` with 24h TTL. If stale/missing, fetch `https://api.scryfall.com/sets` with `User-Agent: Counterflux/1.0` header. Filter to relevant set types: `expansion`, `commander`, `masters`, `core`, `draft_innovation`, `funny`. Exclude sets with `parent_set_code` (commander decks, sub-products). Cache in meta table as `{ key: 'scryfall-sets', data: [...], updated_at: Date.now() }`. Return array of set objects.
- `getCachedSets()` -- synchronous read from in-memory cache (populated on first fetchSets call). Returns `[]` if not yet loaded.

Each set object shape: `{ code, name, set_type, released_at, icon_svg_uri, card_count }`.

**5. Market store** (`src/stores/market.js`):
Follow the `initCollectionStore()` pattern exactly. Export `initMarketStore()`. The store has:

State:
- `watchlist: []` -- array of watchlist entries from Dexie
- `activeSet: null` -- selected set code for spoiler browser
- `spoilerCards: []` -- cards for current set
- `spoilerFilters: { colours: [], rarity: 'all', type: 'all' }`
- `moversPeriod: '7d'`
- `gainers: []`, `losers: []`
- `sets: []` -- from sets service
- `pendingAlerts: []` -- alerts triggered on load
- `alertBadgeCount: 0`
- `activeTab: 'spoilers'`
- `loading: false`

Methods:
- `async init()` -- load watchlist from db.watchlist.toArray(), call fetchSets(), call snapshotWatchlistPrices(), call checkAlerts(), compute initial movers
- `async addToWatchlist(scryfallId, alertType = null, alertThreshold = null)` -- add entry with `{ scryfall_id: scryfallId, added_at: new Date().toISOString(), alert_type: alertType, alert_threshold: alertThreshold, last_alerted_at: null }`. Enforce unique scryfall_id. Reload watchlist.
- `async removeFromWatchlist(scryfallId)` -- delete from db.watchlist where scryfall_id matches. Reload watchlist.
- `async updateAlert(scryfallId, alertType, alertThreshold)` -- update existing entry
- `async checkAlerts()` -- for each watchlist entry with alert_type set, read current card price from db.cards, convert EUR to GBP using `eurToGbpValue`, compare against threshold. If triggered and not already alerted today, add to pendingAlerts and update last_alerted_at. Set alertBadgeCount.
- `async loadSpoilers(setCode)` -- query `db.cards.where('set').equals(setCode).toArray()`, apply spoilerFilters, set spoilerCards. Set activeSet.
- `filterSpoilers()` -- apply spoilerFilters to loaded spoiler data. Filter by colour (color_identity intersection), rarity, type_line contains.
- `async loadMovers()` -- call computeMovers with moversPeriod, set gainers and losers
- `setTab(tab)` -- set activeTab

**6. Sparkline utility** (`src/utils/sparkline.js`):
Export `renderSparkline(prices, width = 120, height = 32)`. Returns SVG string per the RESEARCH.md Pattern 3. Positive trend (last >= first) uses `#2ECC71` stroke, negative uses `#E23838`. Fill uses linearGradient from stroke colour at 0.15 opacity to transparent. If prices array is empty or length 1, return empty string.
