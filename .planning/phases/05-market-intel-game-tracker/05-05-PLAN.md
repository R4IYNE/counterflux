**1. Life adjuster** (`src/components/life-adjuster.js`):
Export `setupLongPress(button, callback)` -- imperative function that attaches pointer event listeners to a button element.

Implementation per RESEARCH.md Pattern 4:
- Import `getIncrement` from `../utils/game-stats.js`
- On `pointerdown`: record startTime, call callback(1) immediately for single increment, set timeout at 400ms to begin repeating at 200ms intervals using `getIncrement(Date.now() - startTime)` for the value.
- On `pointerup`, `pointerleave`, `pointercancel`: clear timeout and interval.
- Add `touch-action: none` CSS via `button.style.touchAction = 'none'` to prevent browser scroll on mobile.
- Return a cleanup function that removes all event listeners.

**Player card layout:**

  - Poison count (label 700, secondary colour if >= 10, `.lethal-highlight` on the number): small +/- buttons (32x32px)
  - Tax count with computed cost: `TAX: {count} ({cost})` (label 400). Small +/- buttons.

  - Commander damage tracker (rendered via `renderCommanderDamageTracker`)
  - Additional counters section: show any active counters with name + value + +/- buttons

**Player grid:**
- `renderPlayerGrid()`: CSS grid, `grid-cols-2` on >= 768px, `grid-cols-1` on < 768px, gap-md (16px)
- Bottom padding: 80px (space for floating toolbar)
- Iterate `$store.game.players` with `x-for`
- Each cell renders `renderPlayerCard(index)`

**Imperative DOM for long-press:** Since Alpine templates render declaratively but long-press needs imperative event binding, use Alpine `x-init` or `x-effect` to call `setupLongPress` on the button elements after they render. Store cleanup functions and call them on unmount. Example pattern:
```html

        x-init="$nextTick(() => { $el._cleanup = setupLongPress($el, (amt) => $store.game.adjustLife(playerIndex, amt)); })"
        x-destroy="$el._cleanup?.()">
```
If `x-destroy` is not available in Alpine 3.15, use the screen cleanup pattern (container._cleanup) to clean up all long-press handlers.
