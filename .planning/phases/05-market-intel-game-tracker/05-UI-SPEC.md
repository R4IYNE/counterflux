# Phase 5 — UI Design Contract

> Visual and interaction contract for Market Intel (Preordain) + Game Tracker (Vandalblast). Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

Inherited from Phase 1. No changes.

| Property | Value |
|----------|-------|
| Tool | none (vanilla JS + Alpine.js) |
| Preset | not applicable |
| Component library | none (custom components, Organic Brutalism) |
| Icon library | Material Symbols Outlined (FILL 0, wght 400, opsz 24) |
| Specialty icon fonts | mana-font (mana symbols), keyrune (set symbols) |
| Display font | Syne (self-hosted .woff2, 400/700) |

| CSS framework | Tailwind CSS v4 (CSS-first config via `@theme`) |
| Border radius | 0px everywhere (Organic Brutalism) |

**Source:** Phase 1 UI-SPEC, no overrides.

## Spacing Scale

Inherited from Phase 1. No changes.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, badge offsets |
| sm | 8px | Compact element spacing, player card internal padding |
| md | 16px | Default element spacing, tab gaps, card-to-card spacing |
| lg | 24px | Section padding, player card content separation |
| xl | 32px | Layout column gaps, main content padding |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level vertical spacing |

Exceptions:
- **Game Tracker touch targets**: Life total +/- buttons are 48x48px minimum for mobile tap accuracy (uses 2xl token). This is not an exception to the scale -- it uses 48px from the standard scale.
- **Life total display**: Life total numbers use 64px (3xl) vertical space allocation for readability at arm's length.

**Source:** Phase 1 UI-SPEC spacing scale. Touch target sizing per GAME-02, GAME-13.

## Typography

Inherited from Phase 1. No changes to the scale.

| Token | Size | Font | Weight | Line Height | Letter Spacing | Transform | Usage |
|-------|------|------|--------|-------------|----------------|-----------|-------|
| display | 48px | Syne | 700 | 1.1 | -0.02em | none | Life totals (primary number), portfolio value in watchlist summary |
| heading | 20px | Syne | 700 | 1.2 | 0.01em | none | Tab section titles, player names, post-game summary heading |
| body | 14px | Space Grotesk | 400 | 1.5 | normal | none | Card oracle text, price descriptions, game notes, Mila messages |
| label | 11px | JetBrains Mono | 400 or 700 | 1.3 | 0.15em | uppercase | Counter labels, stat headers, timestamps, button text, tab labels |

### Phase 5 Specific Typography Notes

- Life total numbers use `display` tier (48px Syne 700) for at-a-glance readability during gameplay
- Commander damage sub-numbers use `heading` tier (20px Syne 700)
- Poison/tax/counter values use `label` tier at weight 700
- Sparkline axis labels (if any) use `label` tier at weight 400
- Price values in watchlist use `label` tier at weight 700, with currency symbol in weight 400
- Tab labels use `label` tier at weight 700 for active tab, weight 400 for inactive

**Source:** Phase 1 UI-SPEC typography scale.

### Semantic Colours (Phase 5 Usage)

| State | Colour | Phase 5 Usage |

### Player Colour Assignments (Game Tracker)

Each player in a game needs a distinguishable colour for the life chart lines and player card accent. Use the existing palette plus MTG colour-inspired tones:

| Player Slot | Colour | Hex | Rationale |
|-------------|--------|-----|-----------|
| Player 1 (you) | Primary Blue | #0D52BD | Matches app accent |
| Player 2 | Red | #E23838 | Matches app secondary |
| Player 3 | Green | #2ECC71 | Matches success token |
| Player 4 | Amber | #F39C12 | Matches warning token |
| Player 5 | Purple | #A855F7 | Distinct from existing palette |
| Player 6 | Cyan | #22D3EE | Distinct from existing palette |

These colours are used ONLY for:
- Life chart line colours (Chart.js dataset colours)
- Thin left border accent on player cards (4px, using ghost-border pattern)
- Commander damage "from" indicators

## Interaction Patterns (Phase 5 Scope)

### Preordain Navigation

| Interaction | Behaviour |
|-------------|-----------|
| Click tab | Swap tab content. Active tab: text-primary, 2px primary bottom border, glow-blue shadow. Inactive: text-muted, no border. No transition animation between tab content (instant swap). |
| Change set in spoiler browser | Dropdown selection reloads gallery with new set data. Loading skeleton (ghost border rectangles) shown during fetch. |
| Click card in spoiler gallery | Opens existing card detail flyout. |
| Right-click card tile | Existing context menu + new "Watch Price" option. |

### Watchlist

| Interaction | Behaviour |
|-------------|-----------|
| Add card to watchlist | Via context menu "Watch Price", card flyout "Watch" button, or search-add within watchlist tab. Card appears at top of list. Toast confirms: `{name} added to watchlist.` |
| Remove from watchlist | Click remove icon (X) on row. Inline undo toast (5s). No confirmation modal. |
| Edit alert threshold | Click threshold value to inline edit. Input appears in-place. Enter or blur confirms. Escape cancels. |
| Sparkline period toggle | Click 7D/30D/90D. Sparkline redraws with selected period data. Active period: text-primary. Inactive: text-dim. |
| Price alert triggers | On app load, market store checks watchlist prices against thresholds. Triggered alerts: toast notification + sidebar Preordain icon gets red dot badge + topbar bell badge count increments. |

### Game Setup

| Interaction | Behaviour |
|-------------|-----------|
| Select deck | Dropdown shows user's decks (from deck store). Selection populates commander automatically. Or skip deck and type commander name with autocomplete. |
| Add opponent | Click "Add Opponent". New row appears with name input + commander input. Max 5 opponents (6 players total per D-01). |
| Remove opponent | Click X on opponent row. Instant removal, no confirmation. |
| Start game | Validates: deck/commander required, at least 1 opponent. Creates game in store, transitions to active game view. |

### Active Game

| Interaction | Behaviour |
|-------------|-----------|
| Tap +/- on life | Instant update. Number scales 1.1x briefly (150ms). Life change logged for chart. |
| Long-press +/- | After 1s hold: increment by 5 every 200ms. After 2s hold: increment by 10 every 200ms. Release stops. Per D-02. |
| Tap player card | Toggles expanded state. Expanded: card height increases, reveals commander damage trackers, additional counters, tax. Smooth height transition (200ms). |
| Next Turn | Turn counter increments. Timer resets/pauses. Turn change logged for chart x-axis. |
| End Game | Modal confirmation. On confirm: transition to post-game overlay. Game data frozen. |

### Dice/Coin

| Interaction | Behaviour |
|-------------|-----------|
| Tap dice icon | Popover opens above toolbar. Select die type, then roll. Result animates in (scale 0.8 -> 1.2 -> 1.0, 400ms). Result fades after 3s. |
| Tap coin icon | Popover opens. Single flip button. Result animates with Y-axis rotation (300ms). |
| High roll mode | Toggle in dice popover. When active, all players roll simultaneously. Highest shown with primary highlight. |

### Post-Game

| Interaction | Behaviour |
|-------------|-----------|
| Mark winner | Tap player name in overlay. Selected player highlighted with primary border. Can change before saving. |
| Mark eliminations | Tap players in elimination order (1st eliminated, 2nd, etc.). Numbers appear next to names. Tap again to remove. |
| Save & Close | Saves game to IndexedDB. Toast: `Game saved to history.` Returns to game setup view. |
| Discard | Modal confirmation. Discards all game data. Returns to game setup view. |

### Game History

| Interaction | Behaviour |
|-------------|-----------|
| Tap past game | Reopens post-game overlay in read-only mode (no Save button, just Close). |
| Delete game | Swipe-to-delete on mobile, X icon on desktop. Inline undo toast (5s). |

### Mobile Responsive (Vandalblast Only)

| Breakpoint | Layout Change |
|------------|--------------|
| >= 768px | 2x2 player grid, toolbar at bottom, stats cards in horizontal row |
| < 768px | Single column player stack, toolbar still at bottom, stats cards stack vertically, touch targets remain 48px minimum |
| < 768px | Game setup form goes full-width, opponent cards stack |

Preordain is desktop-only (no mobile-responsive requirement per project constraints).
